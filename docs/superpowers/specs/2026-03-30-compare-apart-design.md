# Compare Apart - 아파트 비교 서비스 설계

## 개요

보유 자금(9.5억~11.5억) 범위 내에서 구매 가능한 서울/경기 아파트 매물을 리스트업하고, 주요 기준에 따라 비교할 수 있는 웹 페이지. 데이터는 네이버 부동산에서 수집하여 Excel로 관리하고, 정적 웹 페이지로 시각화한다. 가족과 함께 보며 의사결정하는 용도.

## 아키텍처

```
[Excel 파일] → [Python 변환 스크립트] → [apartments.json]
                                              ↓
                              [정적 웹 프론트엔드 (HTML/CSS/JS)]
                                              ↓
                              [로컬 FastAPI 서버 / GitHub Pages]
```

- **데이터 관리**: Excel (.xlsx) 파일로 수동 관리
- **데이터 변환**: Python 스크립트로 Excel → JSON 변환
- **웹 프론트엔드**: 순수 HTML/CSS/JS (프레임워크 없음)
- **로컬 개발**: FastAPI로 정적 파일 서빙
- **배포**: GitHub Pages (web/ 폴더만 배포)

## 대상 매물 조건

| 조건 | 값 |
|------|-----|
| 매매가 범위 | 9.5억 ~ 11.5억 |
| 면적 타입 | 59타입, 84타입, 그 이하 |
| 세대수 | 300세대 이상 |
| 지역 | 서울, 경기 |
| 층수 제한 | 5층 이하 / "저층" 표기 매물은 매매가 범위 계산에서 제외 (목록에는 흐리게 표시) |

## 비교 기준 (우선순위)

1. **강남역까지 교통** (최우선) — 소요시간, 교통수단
2. **최근역까지 거리** — 역명, 도보 시간
3. **주변환경** — 대형마트 유무/거리, 상권(상가 밀집도), 혐오시설 유무
4. **주변학교** — 초/중/고
5. **면적** — 전용면적, 타입
6. **금액** — 매매가 범위 (저층 제외)

## 데이터 변환 규칙 (convert.py)

1. Excel 파일 읽기 (openpyxl)
2. 300세대 미만 단지 제외
3. 매매가 9.5억~11.5억 범위 밖 매물 제외
4. 5층 이하 또는 "저층" 텍스트 포함 매물에 `is_low_floor: true` 플래그
5. 같은 단지+타입으로 그룹핑
6. 매매가 범위 계산 시 저층 매물 제외
7. 지도 이미지 파일을 `web/maps/`로 복사
8. `web/apartments.json`으로 출력

## 웹 UI 구조

### 페이지 구성

단일 페이지 (index.html)

### 상단: 필터바

현재 적용 조건 표시 (9.5억~11.5억 / 59·84타입 / 300세대+ / 서울·경기) 및 정렬 옵션.

### 메인: 매물 리스트 (카드)

각 카드에 표시되는 정보:
- 순위 번호
- 단지명, 지역, 타입, 세대수
- 강남역 소요시간 (색상 구분: 가까울수록 초록, 멀수록 주황)
- 매매가 범위 (저층 제외)
- 매물 건수

### 드롭다운 상세 (클릭 시 펼침)

2분할 레이아웃:
- **좌측**: 교통, 단지정보, 주변환경, 학교 (2x2 그리드)
- **우측**: 네이버 지도 대중교통 경로 스크린샷 (560x440px)

하단에 개별 매물 테이블:
- 층수, 매매가, 네이버 부동산 설명
- 저층 매물은 흐리게 표시 + "저층" 뱃지, 매매가 범위 미포함 안내

### 정렬 옵션

- 기본: 강남역 가까운 순
- 매매가 낮은순 / 높은순
- 세대수순
- 면적타입별

## 프로젝트 구조

```
compare-apart/
├── data/
│   ├── apartments.xlsx          # 원본 데이터 (수동 관리)
│   └── maps/                    # 네이버 지도 스크린샷 원본
│       └── raemian_wonbailey_map.png
├── scripts/
│   └── convert.py               # Excel → JSON 변환
├── web/
│   ├── index.html               # 메인 페이지
│   ├── style.css                # 스타일
│   ├── app.js                   # 인터랙션 (드롭다운, 정렬)
│   ├── apartments.json          # 변환된 데이터
│   └── maps/                    # 이미지 (배포용 복사본)
├── server.py                    # 로컬 개발용 FastAPI 서버
├── requirements.txt             # Python 의존성
└── .gitignore
```

## 데이터 소스

- **매물 정보**: 네이버 부동산에서 수동 수집 → Excel에 입력
- **교통 경로**: 네이버 지도 길찾기에서 대중교통 경로 스크린샷 캡처 → `data/maps/`에 저장

## 배포

- **로컬**: `server.py` 실행으로 FastAPI 서버 기동, 같은 네트워크에서 가족 접속 가능
- **배포**: `web/` 폴더를 GitHub Pages로 배포 (정적 파일만)
